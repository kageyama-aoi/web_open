1. 勉強会の全体構成（90分）

タイムライン

時間内容狙い
10分オープニング＆アイスブレイク安心感・ゴール共有
10分AIとの付き合い方（会話劇）期待値調整・対話のコツ
10分プロンプトの型を知る整理型／会話型を学ぶ
15分実演① スプシ関数の調べ方“やりたいこと”→関数
15分実演② 未知の知識の調べ方条件設計＆公式確認
20分ワークショップ自分事化・定着
10分まとめ・次の一歩行動宣言

ゴール

  • AIへの質問の型（目的＋条件＋形式）を掴む
  • スプシ関数・調べものを実務で即試せる状態
  • 「今週AIに1回質問する」行動の第一歩
進行のコツ：専門用語は避け、手を動かす→振り返りの順で安心感をキープ。

3. 会話劇：職場シーン風の台本作成例

登場人物

  • 課長：現場をまとめる立場。新しいツール導入には慎重。
  • 久保：現場担当の新人。AIを少し試したことはあるが半信半疑。フレンドリーで距離が近い。
  • 篠原：AIに詳しい社員。具体的な使い方を紹介してリード役。

会話劇の例

課長「最近AIを業務に活かそうって話が出てるけど、正直うちには向かない気がするな。」

久保「わかりますよ、課長。この前試したら答え間違ってて、全然ダメだったっす。」

篠原「久保さん、それなら“それ間違ってるよ”って突っ込めばいいんです。AIなら怒らないし、修正してくれますよ。」

課長「人間相手だと角が立つけど、AIなら突っ込めるってわけか。」

久保「でも篠原さん、答えがちょっと物足りなくないっすか？ 現場で使うにはもう少し詳しくほしいっす。」

篠原「そういうときは“そっちの方向性じゃない。もっと具体的に”って突っ込めば、追加で具体的な情報を出してくれます。」

課長「でも最新情報が出てこないのは困るな。法改正やシステム変更が絡むからな。」

篠原「たとえば“これ古くない？ 2024年以降の情報を探して”って指定すれば精度が上がります。課長、それで足りなければ検索や公式資料を組み合わせれば十分です。」

久保「なるほどっす。でも、ついAIに万能の先生を期待しちゃうんすよね。」

篠原「AIは“突っ込んで調整できる派遣スタッフ”くらいに思うのがちょうどいいんです。」

久保「でも、どう聞けばいいか分からないんすよ。言い方が難しくて。」

篠原「そこもAIに頼めばいいんです。“この質問の仕方を直して”とか、“議事録をまとめるのにいいお願いの言い方を作って”って頼めば、雛形を作ってくれますよ。」

課長「へぇ、言い方そのものもAIに考えさせればいいのか。」

篠原「例えば“議事録をまとめるのにいいお願いの言い方を作って”って頼むと――」

久保「AIはどう返すんすか？」

篠原「“決定事項、議論の経緯、アクション、次回課題の4つに整理してMarkdownで出力してください”みたいに、すぐ使える形を返してくれます。」

課長「なるほど…。ゼロから考えなくて済むのは助かるな。」

課長「ただ、答えに近づくまで何度もやりとりするのは面倒だな。」

篠原「課長、それは発想を変える必要があります。“違う”“もっと掘り下げて”と突っ込む前提で会話する。1回で完璧を求めるより、磨き上げた方が早いんです。」

久保「でも課長、業務データは機密だから使えないっすよね。」

篠原「本番データはNGです。ただ“マニュアルの下書き”“会議メモの整理”“説明資料の雛形”なら安心して使えます。そこから始めるのが現実的です。」

課長「なるほど…。周辺業務からならリスクも低いし効率化もできそうだな。」

久保「確かに、“使えない”と思ってたのは、期待の仕方を間違えてただけかもしれないっす。」

篠原「そうなんです。AIは万能じゃない。でも“突っ込んで調整できる派遣スタッフ”としては優秀。周辺業務からでも導入すれば、現場はかなり楽になりますよ。」

コラム：実務で使える「突っ込み」例

  • 最新情報を指定したいとき
    「これ古くない？ 2024年以降の情報を探して」
    👉 年を入れると精度アップ
  • 一般論で終わったとき
    「そっちの方向性じゃない。現場で使えるレベルまで具体的に」
  • 説明が長すぎるとき
    「要点だけで。3行以内でまとめて」
  • 判断根拠が欲しいとき
    「その答えの根拠や出典を教えて」
4. プロンプトの工夫（入口体験：整理型／会話型）

1A. 整理型（目的・条件・形式で書く）

しっかり言語化してから依頼したい人向け。後で資料にも残しやすい。

【目的】◯◯をしたい
【対象】中学生向け／社会人向け／初心者向け
【形式】会話形式／表形式／箇条書き
【条件】例を3つ／公式URLを添えて／よくある注意点も
【質問】＿＿＿＿＿＿＿＿＿＿＿＿＿＿＿＿＿＿＿＿
例（スプシ関数）：
目的日付から曜日を日本語の短縮形で表示したい
形式ステップ→関数→注意点（箇条書き）
質問Googleスプレッドシートで、A1の日付の曜日を日本語の短縮形で表示する関数と注意点を、ステップ＋関数例＋よくあるミスの順で教えて。

1B. 会話型（先生と生徒のやりとり）

“目的とか書くのめんどくさい派”はこちら。会話で進めて理解する。

会話形式で説明してください。
あなたは先生役、私は生徒役です。
スプレッドシートで日付から曜日を出す方法を、質問と答えのやり取りで教えてください。
助成金について説明してください。
社会人向けに、3分で読める要点だけ。表形式で「制度名／対象／金額／URL」を整理してください。

※ 会話型で理解 → 必要に応じて整理型に「転記」すると再利用が楽。

整理型 vs 会話型（対応表）

スタイル特徴向いている人例プロンプト
整理型 目的・条件を表で整理／後から再利用しやすい 業務に残す・反復利用したい 「【目的】◯◯ 【条件】◯◯ 【形式】◯◯」
会話型 対話で自然に理解／気楽に始められる 最初の学習・文章化が苦手 「先生役と生徒役で説明して」
おすすめ進行：まず全員で会話型を体験 → 整理型へ写経して“型”を掴む。

スライド5：AI質問の黄金ルール
  1. 目的＋条件＋形式で聞く
  2. 最初は大まか→徐々に具体化
  3. 条件は漏れなく短く（箇条書き歓迎）
  4. 最後は公式情報で確認
例）「Googleスプレッドシートで、A1の日付から曜日を日本語の短縮形で表示する関数と注意点を教えて。
形式はステップ＋関数例＋よくあるミスでお願いします。」
スライド6：実演① スプシ関数の調べ方

例題：「日付から曜日を日本語で出すには？」 → =TEXT(A1,"aaaa")

  • 質問を“やりたいこと”起点で作る
  • 引数・注意点・代替案（TEXT/TEXTJOIN等）も一緒に聞く
  • サンプル表に関数を入れて実際に動作確認
スライド7：実演② 未知の知識（助成金）を“自分用”に引っ張る
  1. 全体像：「日本の小規模事業者向け助成金の種類を表で」
  2. 条件追加：「東京都／従業員2名／機材・広告OK」
  3. 公式URL：「必ず募集要項のURL併記」
  4. 比較：「金額・採択率・手軽さで比較表」
ポイント：条件設計で“自分にフィット”が決まる。
スライド8：ワークショップ
  1. ペアで関数をAIに聞いてスプシへ実装
  2. 未知の制度（例：助成金）を各自条件で検索
  3. 実務アイデアを1つ書き出し、1分共有

司会の動き：質問文の添削と“条件の言語化”支援に集中。

スライド9：まとめ・次のステップ
  • 今日の学びを1人1行で共有（チャットor口頭）
  • 宿題：「今週AIに1回質問」→テーマをその場で宣言
  • 次回予告：「GASで作る簡単自動化（メール下書き／定型処理）」

話し方メモ（隊長向け）

  • 最初に「正解は1つじゃない」前提を置く
  • 実演→手元で再現→気づき共有の3拍子
  • わからなさは“設計不足”。条件を一緒に言語化
この資料は、初心者が「最初の成功体験」を得やすい順番に構成しています（会話で入口→型で整理→実演→ワーク→宣言）。
なぜこの再構成か：最初に“聞き方”を体験して不安を下げ、次に“型”で定着させると、その後の実演（関数／助成金）にスムーズに接続できるからです。
